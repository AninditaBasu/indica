The Vedic APIs

All of the data in the Vedas, presented as JSON or XML objects through APIs

These APIs make available to you two kinds of data: data regarding the gods, poets, and poem meters used in all of the Rig Veda verses, and data regarding the descriptions of all (almost) nouns mentioned in the entire vedic literature. Here are two examples:

[
    {
        "mandal": 1,
        "sukta": 2,
        "sungby": "Madhuchchhanda Vaishwamitra",
        "sungfor": "Vayu",
        "meter": "Gayatri"
    }
]
     
[
    {
        "word": "aksh",
        "nagari": "अक्ष",
        "description": "the axle of a chariot",
        "category": "chariot"
    }
]

The data is returned as JSON or XML objects. You can programmatically process the data to make visually appealing or easily consumable information. Some use case ideas are listed at What can I do with this data?

Making requests to the APIs

The following APIs are available:

  • rv: Returns the gods, poets, and meters from all of the Rig Veda verses.
  • vs: Returns the descriptions of all nouns (names, places, animals, things) from the entire vedic literature.

Authentication

Not needed. These are open APIs.

The APIs are hosted on Sheetlabs, which limits the monthly API calls (for free accounts like mine) to 10,000. This limit includes all of us making all of our calls to all of these APIs. Because the APIs need no authentication, I am neither tracking who makes how many calls nor throttling calls on IP-address basis. Just use the APIs in a fair manner, is all that I can say.

More about these APIs is at About the Vedic APIs.

Getting started

Endpoints

JSON output is returned by the following endpoints:

  • https://sheetlabs.com/RV/rv, for gods, poets, and meters.
  • https://sheetlabs.com/RV/vs, for descriptions of nouns.

To get output in the XML format, append .xml to the endpoint, like this: https://sheetlabs.com/RV/rv.xml.

Resources

The following resources are available:

  • rv: mandal, sukta, sungby, sungfor, and meter
  • vs: word, nagari, description, and category

The mandals and suktas of the rv API are integers; the remaining parameters of the rv API and all of the parameters of the vs API are strings.

For details of the API endpoints and resources, see rv API reference documentation and vs API reference documentation.

Filtering

Optional. All of the parameters can be used for filtering except the nagari parameter of vs API.

You can specify a range for the integers, or partial text for the strings. You can supply more than one parameter in a single call by separating them with an ampersand (&), for example, https://sheetlabs.com/RV/rv?mandal=10&meter=gayatri&sungby=*dra.
