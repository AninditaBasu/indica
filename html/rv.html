Vedic APIs 1.0

rv API reference

All of the poets and gods in the Rig Veda, presented with the mandal and sukta numbers, and the meters the verses are composed in.

You retrieve the data by making GET calls to its REST resource namespaces.

Endpoint

https://sheetlabs.com/RV/rv

Resources

The GET calls are served over HTTPS. The following resources are available:

  • mandal: The mandal number for the verse. For example, 7.
    This is an integer resource. Filtering is allowed, and you can use ranges, for example, 1..2 returns all of the verses in the first and second mandals.
    Valid values for this resource are 1 through 10 (because there are only 10 mandals in the Rig Veda).
  • sukta: The sukta number for the verse. For example, 136.
    This is an integer resource. Filtering is allowed, and you can use ranges, for example, sukta=10..20 returns all of the suktas numbered between 10 and 20 (both inclusive) from all of the 10 mandals.
    The number of suktas in each mandal is different; for example, mandal 1 contains ... suktas. The highest value possible for this resource is ... (which is the number in the ... mandal).
  • sungby: The composer of the verse. For example, Vishwamitra.
    This is a string resource. Filtering is allowed, and partial values can be used. For example *aindra returns all of the verses composed by anyone whose name ends with aindra.
  • sungfor: The god, person, or thing that the verse is composed for. For example, Agni or plough.
    This is a string resource. Filtering is allowed, and partial values can be used.
  • meter: The meter that the verse is composed in. For example, Trishtubh.
    This is a string resource. Filtering is allowed, and partial values can be used.

To filter results, you can use more than one resource, separated by an ampersand (&), for example, https://sheetlabs.com/RV/rv?mandal=10&meter=gayatri&sungby=*dra.
